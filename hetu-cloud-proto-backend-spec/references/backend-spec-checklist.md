# 后端需求说明自检清单

- 是否先输出了《冲突说明（Q&A）》并等用户回答后再写最终说明？
- 是否检查过 Q&A 里不存在空的 `你的回答：`？若存在，是否已追问补齐或明确标记为 TBD？
- 是否在最终文档中包含 “冲突 Q&A 结论汇总（Decision Log）”，并覆盖所有关键问题（Q1/Q2/...）？
- 是否把“术语/对象”统一了（同一概念不再多名）？
- 是否有完整的角色、权限点、数据范围说明？
- 是否明确了唯一性、幂等策略、重复提交处理？
- 是否把所有枚举值列全，并给出含义与来源？
- 是否有状态机/流转图或等价描述（允许/禁止路径）？
- 是否把关键校验写成可实现的规则（字段校验 + 跨表校验）？
- 是否明确了删除策略与级联影响（含关联同步）？
- 接口清单是否覆盖：列表/详情/新增/编辑/删除/关键动作接口？
- 是否给出关键错误码/错误文案建议（尤其是禁止路径）？
- “重点难点”是否真的点到了后端最容易翻车的点？
- 是否统一了字段命名（同一概念同名，比如 items/storageItems、updateTime/updatedAt 等）？
- 是否把默认假设与 TBD 统一收敛到“未决项”？
