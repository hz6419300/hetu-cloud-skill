# 冲突说明（Q&A）模板

> 目的：把会影响后端实现的冲突/缺口一次性问清楚，用户逐条回答后再写最终后端需求说明。  
> 输出原则：问题要短、影响要明确、回答要能直接落到“数据/校验/状态/接口/权限”。  
> 回答提醒：如果该点明确是 UI 纯交互/样式，可以直接写“前端处理/out-of-scope”；但只要涉及接口/数据/校验/删除/验收口径，请把后端要做/不做什么说清楚（是否落库、是否校验、接口是否透传、默认值与兜底、错误提示）。

## Q&A

### Q1

- 问题：
- 上下文/依据：（引用原文短句，或注明小节/页面）
- 为什么问（影响）：（会影响哪些表/字段/状态/接口/权限/验收）
- 建议默认（可选）：（如果用户不回答，默认按什么做；必须写清楚是假设）
- 你的回答：（建议第一句先给“可落地结论”，再补充说明）
