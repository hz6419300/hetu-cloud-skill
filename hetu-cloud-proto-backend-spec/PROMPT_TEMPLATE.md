## 2) 生成最终后端需求说明阶段

基于“原始需求 + 我已填写的冲突 Q&A”，请生成最终《后端需求说明》Markdown，要求：

- 先检查冲突 Q&A 里是否还有空的 `你的回答：`；如有，先列出未回答问题并要求我补齐，或将其明确标记为 TBD 再继续输出。

- 所有默认假设与 TBD 统一收敛到文末的“未决项”。
- 对每条关键规则给出可验收口径（允许/禁止路径、错误提示建议）。
